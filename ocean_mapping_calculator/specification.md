# 海洋マッピング座標計算ツール 仕様書

## 1. 概要

本仕様書は、海洋上または海中のマッピング地点について、基準点からの相対座標を計算し、地図上に可視化する単一HTMLアプリケーションの仕様をまとめたものである。

本アプリケーションは、以下の情報を入力として受け取る。

- 基準点との距離
- マッピング地点の深度
- 方角
- 方角の意味
- 距離の種類
- ポイント名称
- ポイントアイコン

計算結果として、基準点を原点とした `x, y, z` 座標をメートル単位で算出する。

---

## 2. 前提条件

### 2.1 基準点

基準点は海抜 0m の地点とし、座標原点として扱う。

```text
基準点 = (x, y, z) = (0, 0, 0)
```

### 2.2 座標系

| 軸 | 正方向 | 負方向 | 単位 |
|---|---|---|---|
| x | 東 | 西 | m |
| y | 北 | 南 | m |
| z | 海面より上 | 海面より下 | m |

海中の深度は、z座標では負の値として扱う。

```text
z = -深度
```

### 2.3 方角の定義

| 方角 | 意味 |
|---:|---|
| 0° | 北 |
| 90° | 東 |
| 180° | 南 |
| 270° | 西 |

## 4. 方角変換仕様

### 4.1 入力方角が「基準点からマッピング地点への方角」の場合

入力された方角をそのまま計算に使用する。

```text
bearing_from_base = bearing_input
```

### 4.2 入力方角が「マッピング地点から基準点への方角」の場合

入力方角を180°反転して、基準点からマッピング地点への方角に変換する。

```text
bearing_from_base = (bearing_input + 180) mod 360
```

例：

```text
入力方角 = 268°
方角の意味 = マッピング地点から基準点への方角

bearing_from_base = (268 + 180) mod 360
                  = 88°
```

---

## 5. 距離計算仕様

### 5.1 距離が水平距離の場合

入力された距離をそのまま水平距離として使用する。

```text
horizontal_distance = distance
```

### 5.2 距離が斜距離の場合

ソナー距離、ケーブル長、対象点までの直線距離などを想定する。

```text
horizontal_distance = sqrt(distance^2 - depth^2)
```

ただし、以下の場合は不正入力とする。

```text
distance < depth
```

理由：

```text
斜距離は深度以上である必要があるため
```

---

## 6. 座標計算仕様

### 6.1 角度のラジアン変換

JavaScriptの三角関数はラジアンを使用するため、方角をラジアンに変換する。

```text
rad = bearing_from_base * PI / 180
```

### 6.2 x座標

x座標は東西方向の位置を表す。

```text
x = horizontal_distance * sin(rad)
```

- x > 0: 東
- x < 0: 西

### 6.3 y座標

y座標は南北方向の位置を表す。

```text
y = horizontal_distance * cos(rad)
```

- y > 0: 北
- y < 0: 南

### 6.4 z座標

z座標は深度を表す。

```text
z = -depth
```

## 7. 計算例

### 7.1 例1

入力：

| 項目 | 値 |
|---|---:|
| 距離 | 1400m |
| 深度 | 178m |
| 方角 | 275° |
| 方角の意味 | マッピング地点から基準点への方角 |
| 距離の種類 | 斜距離 |

方角変換：

```text
bearing_from_base = (275 + 180) mod 360
                  = 95°
```

水平距離：

```text
horizontal_distance = sqrt(1400^2 - 178^2)
                    ≒ 1388.64m
```

計算結果：

| 項目 | 値 |
|---|---:|
| x | 約 +1383.35m |
| y | 約 -121.03m |
| z | -178m |

意味：

```text
基準点から見て、東へ約1383.4m、南へ約121.0m、深さ178m
```

### 7.2 例2

入力：

| 項目 | 値 |
|---|---:|
| 距離 | 1291m |
| 深度 | 128m |
| 方角 | 268° |
| 方角の意味 | マッピング地点から基準点への方角 |
| 距離の種類 | 斜距離 |

方角変換：

```text
bearing_from_base = (268 + 180) mod 360
                  = 88°
```

水平距離：

```text
horizontal_distance = sqrt(1291^2 - 128^2)
                    ≒ 1284.64m
```

計算結果：

| 項目 | 値 |
|---|---:|
| x | 約 +1283.86m |
| y | 約 +44.83m |
| z | -128m |

意味：

```text
基準点から見て、東へ約1283.9m、北へ約44.8m、深さ128m
```

## 8. 画面仕様

### 8.1 入力エリア

### 8.2 計算結果エリア

計算結果エリアには以下を表示する。

- x座標
- y座標
- z座標
- 方角変換後の値
- 水平距離
- 計算式
- 方角コンパス（SVG）
- テキストによる位置説明

### 8.3 マッピングエリア

マッピングエリアでは、計算したポイントをSVG地図上に表示する。

表示内容：

- 基準点
- x軸・y軸（南北・東西方向の主軸線）
- 東西南北ラベル
- グリッド（3段階：細線 / 太線 / 数字ラベル）
- 登録済みポイント（絵文字アイコン＋名称）
- 現在の計算結果のプレビュー表示（点線）
- ステータスバー（表示範囲・グリッド間隔・操作説明）

#### マップ操作

| 操作 | 動作 |
|------|------|
| マウスホイール | ズームイン / アウト（カーソル位置中心） |
| ドラッグ | パン（移動） |
| タッチ1本指 | パン |
| タッチ2本指（ピンチ） | ズームイン / アウト |
| 「地図を再描画」ボタン | ズーム・パンをリセットして全体表示 |

#### グリッド仕様

| 種類 | 基本間隔 | 自動拡大条件 |
|------|----------|-------------|
| 細線（目盛り） | 10m | ピクセル幅 < 3px で2倍 |
| 太線（格子） | ラベルの1/2 | ラベル間隔に連動 |
| 数字ラベル | 200m | ピクセル幅 < 40px で2倍 |

スケールが小さくラベルが重なる場合は、自動的に間隔を拡大する。

### 8.4 計算履歴エリア

計算履歴エリアには、登録済みポイントを表形式で表示する。

表示項目：

- ポイント名称（アイコン・色ドット付き）

- 深度

- x
- y
- z

## 9. 機能仕様

### 9.1 計算機能

「計算する」ボタンまたは入力変更により、現在の入力値をもとに座標を計算する。

計算結果は以下に反映する。

- 数値結果
- 計算式
- 方角コンパス
- マッピングプレビュー
- コピー用テキスト

### 9.2 履歴と地図へ追加機能

「履歴と地図へ追加」ボタンを押すと、現在の計算結果を履歴に追加する。

追加されたポイントはマッピングエリアに固定表示される。  
ポイントカラーは登録順にパレットから自動割り当てされる。

### 9.3 ポイント削除機能

履歴テーブルの削除ボタンを押すことで、該当ポイントを履歴および地図から削除する。

### 9.4 履歴全削除機能

「履歴を全削除」ボタンを押すことで、登録済みの全ポイントを削除する。

削除前に確認ダイアログを表示する。

### 9.5 結果コピー機能

「結果をコピー」ボタンを押すことで、現在の計算結果をクリップボードへコピーする。

コピー対象には以下を含む。

- ポイント名称
- アイコン
- 色
- 距離
- 深度
- 入力方角
- 基準点から対象地点への方角
- 水平距離
- x
- y
- z
- 日本語による位置説明

### 9.6 CSV保存機能

「CSV保存」ボタンを押すことで、現在の計算履歴をCSVとして保存する。

ファイル名形式：

```text
ocean_mapping_history_YYYY-MM-DDTHH-MM-SS.csv
```

CSVはExcel等で開きやすいよう、UTF-8 BOM付きで出力する。

### 9.7 CSV読み込み機能

「CSV読み込み」ボタンを押すことで、過去に保存したCSVを読み込む。

読み込んだポイントは、現在の履歴に追加される。

### 9.8 マップ パン機能

マウスドラッグまたはタッチ1本指操作でマップを移動（パン）できる。  
SVGのviewBox属性を変更することで実現する。

### 9.9 マップ ズーム機能

マウスホイールまたはタッチ2本指ピンチ操作でマップをズームできる。  
カーソル（またはピンチ中心）位置を基点としてズームする。

ズーム範囲：

```text
最大ズームイン：初期viewBox幅の 1/12
最大ズームアウト：初期viewBox幅の 12倍
```

「地図を再描画」ボタンでパン・ズームをリセットして全ポイントが見える全体表示に戻る。

## 11. バリデーション仕様

### 11.1 距離

```text
distance >= 0
```

### 11.2 深度

```text
depth >= 0
```

### 11.3 方角

数値である必要がある。

入力値は内部的に以下で正規化する。

```text
bearing = ((bearing % 360) + 360) % 360
```

### 11.4 斜距離の場合

以下を満たす必要がある。

```text
distance >= depth
```

満たさない場合、計算エラーとして表示する。

## 12. 技術仕様

### 12.1 実装方式

本アプリケーションは単一HTMLファイルとして実装する。

構成：

```text
HTML + CSS + JavaScript
```

外部ライブラリは使用しない。

### 12.2 描画方式

マッピング描画にはSVGを使用する。

理由：

- 単一HTMLに組み込みやすい
- 座標軸・グリッド・ラベル描画が容易
- ブラウザ標準機能のみで動作する
- 画像出力や拡張がしやすい

### 12.3 パン・ズームの実装方式

SVGのviewBox属性を動的に変更することでパン・ズームを実現する。  
`<g transform>` は使用しない。

```javascript
// ズーム：viewBoxのw/hを拡縮し、基点位置を維持
// パン：viewBoxのx/yをオフセット
svg.setAttribute("viewBox", `${x} ${y} ${w} ${h}`);
```

### 12.4 データ保持

計算履歴はブラウザ上のJavaScript変数に保持する。

現仕様では、ブラウザを閉じると履歴は消える。

永続化が必要な場合は、以下の拡張を検討する。

- LocalStorage保存
- IndexedDB保存
- JSON保存
- サーバ保存
- GeoJSON出力

## 15. 注意事項

### 15.1 方角の意味に注意

「西向き 268°」などの表現では、その方角が以下のどちらを意味するかを明確にする必要がある。

- 基準点からマッピング地点への方角
- マッピング地点から基準点への方角

本アプリケーションでは選択式で指定可能とする。

### 15.2 距離の種類に注意

入力距離が斜距離か水平距離かで結果が変わる。

- ソナー距離、ケーブル長、直線距離: 斜距離
- 地図上の平面距離: 水平距離

### 15.3 地球曲率について

本仕様の計算は、数百m〜数km程度のローカル座標計算を想定している。

広域測量や高精度な海洋測量では、以下の考慮が必要となる。

- 地球曲率
- 測地系
- 緯度経度
- 投影座標系
- 潮位補正
- 真北・磁北差
- センサー誤差

## 18. まとめ

本アプリケーションは、海洋マッピング作業において、基準点からの相対座標を簡単に算出し、視覚的に管理するためのツールである。

単一HTMLファイルで完結するため、以下の利点がある。

- インストール不要
- オフライン利用可能
- ブラウザのみで動作
- CSVによるデータ受け渡しが可能
- 現場での簡易確認に使いやすい

今後、緯度経度変換やGeoJSON出力、背景地図読み込み、LocalStorage永続化などを追加することで、より本格的な測量・海洋調査支援ツールとして拡張できる。
